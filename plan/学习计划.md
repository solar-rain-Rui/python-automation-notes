### Python自动化测试学习路线 (社招转行版)

此路线分为四个核心阶段，请严格按照顺序和优先级进行，不要跳级。

#### **第一阶段：编程与测试基础 (地基阶段 - 优先级：最高)**

这个阶段的目标是打下坚实的基础。基础不牢，地动山摇。**预计花费全部学习时间的40%**。

| 学习内容 | 技术栈/工具 | 学习要点与优先级 | 备注 |
| :--- | :--- | :--- | :--- |
| **1. Python 编程基础** | `Python 3.x` | **[最高]** 变量、数据类型、流程控制<br>**[最高]** 函数、模块、包<br>**[最高]** **面向对象 (OOP)**：类、对象、继承、封装、多态 (自动化框架的核心思想)<br>**[高]** 文件操作、异常处理 | OOP是重中之重，必须彻底理解，否则无法理解和设计框架。 |
| **2. 软件测试核心理论** | - | **[最高]** 测试流程(STLC)、缺陷生命周期<br>**[最高]** 测试用例设计方法 (等价类、边界值、场景法)<br>**[高]** Web/App测试要点、HTTP协议基础 | 理论是面试必考题，也是你作为“测试”工程师的立身之本。 |
| **3. Linux 基础操作** | `CentOS/Ubuntu` | **[最高]** 常用命令 (ls, cd, cp, mv, rm, grep, ps, top)<br>**[高]** Vim基本使用<br>**[中]** Shell脚本入门 | 自动化脚本最终都在Linux服务器上运行，这是必备技能。 |
| **4. 数据库操作** | `MySQL` | **[最高]** **SQL**：`SELECT`(重点是JOIN)、`INSERT`, `UPDATE`, `DELETE`<br>**[高]** 使用Python库 (`PyMySQL`) 连接和操作数据库 | 测试不仅要看界面，更要验证后端数据的正确性。 |
| **5. 版本控制工具** | `Git` | **[最高]** `clone`, `add`, `commit`, `push`, `pull`, `branch`, `merge`<br>**[高]** 解决基本冲突、了解团队协作流程 | 公司代码都用Git管理，这是团队协作的入场券。 |

---

#### **第二阶段：核心自动化技术栈 (主干阶段 - 优先级：高)**

这是你成为自动化测试工程师的核心技能，**必须逐个攻破**。建议先精通一个方向（推荐Web），再触类旁通。**预计花费全部学习时间的35%**。

| 学习内容 | 技术栈/工具 | 学习要点与优先级 | 备注 |
| :--- | :--- | :--- | :--- |
| **1. Web UI 自动化** | `Selenium`, `Playwright`(新贵) | **[最高]** **元素定位**：XPath、CSS Selector是必须精通的<br>**[最高]** **三大等待**：强制、隐式、显式等待<br>**[最高]** **PO模式 (Page Object Model)**：框架设计的核心<br>**[高]** 浏览器操作、键盘鼠标事件、Cookie处理 | **建议先学Selenium**，市场占有率最高。学通后再了解Playwright，它是未来的趋势，也是你简历的亮点。 |
| **2. 接口自动化** | `requests` 库 | **[最高]** HTTP协议的GET, POST, PUT, DELETE方法<br>**[最高]** 处理请求头(Headers)、请求体(Body/JSON)、响应(Response)<br>**[高]** Cookie和Session处理、Token认证 | 接口自动化是目前招聘需求量最大、投入产出比最高的技能。 |
| **3. 自动化测试框架** | `pytest` | **[最高]** 断言(assert)、Fixture(核心)、setup/teardown<br>**[高]** 参数化 (@parametrize)、Mark标记<br>**[高]** 插件使用：`pytest-html`(报告), `pytest-xdist`(并发) | **放弃unittest，直接学Pytest！** 它是目前Python测试领域的事实标准，功能强大，社区活跃。 |
| **4. App UI 自动化** | `Appium` | **[中]** 环境搭建 (非常重要)<br>**[中]** Desired Capabilities 配置<br>**[中]** 元素定位与交互、手势操作 (滑动、点击) | App自动化和Web自动化原理相通，可以放在Web和接口之后学习，优先级稍低，但也是加分项。 |

---

#### **第三阶段：工程化与进阶 (拉开差距的关键 - 优先级：中)**

掌握了这个阶段的技能，你就不再是一个只会“点点点”的自动化脚本小子，而是具备了“测试开发”的思维。**预计花费全部学习时间的15%**。

| 学习内容 | 技术栈/工具 | 学习要点与优先级 | 备注 |
| :--- | :--- | :--- | :--- |
| **1. 持续集成/部署(CI/CD)** | `Jenkins` | **[高]** 创建Job、配置Git、定时构建、执行自动化脚本<br>**[高]** 查看构建报告、配置邮件通知 | 知道如何将你的自动化脚本集成到公司的发布流程中，这是面试的重要加分项。 |
| **2. 容器化技术** | `Docker` | **[中]** 理解镜像(Image)和容器(Container)的概念<br>**[中]** 常用命令 (`run`, `ps`, `exec`, `logs`)<br>**[中]** Dockerfile入门，能将你的测试环境打包成镜像 | Docker能解决“在我电脑上明明是好的”的经典问题，是现代开发的必备技能。 |
| **3. 性能测试** | `JMeter`, `Locust` | **[中]** 了解核心性能指标 (TPS, 响应时间, 并发数)<br>**[中]** 使用Locust编写简单的Python性能脚本 | 性能测试不是必须，但如果你会，将极具竞争力。建议主学Locust，因为它是Python的。 |

---

#### **第四阶段：项目实战与求职准备 (临门一脚 - 优先级：贯穿始终)**

**这个阶段没有具体的时间限制，应该从第二阶段开始就同步进行。**

1.  **构建个人自动化测试项目 (至关重要！)**
    *   **目标**：选择一个你熟悉的、功能完整的网站或App（例如电商网站、社区论坛）。
    *   **技术整合**：
        *   使用 `pytest` 作为测试框架。
        *   实现Web和接口自动化测试用例。
        *   采用 **PO模式** 组织你的UI测试代码。
        *   通过读取配置文件（如YAML）管理测试数据和环境。
        *   将项目代码托管到 `GitHub` 或 `Gitee`。
        *   **(进阶)** 搭建 `Jenkins` 环境，实现代码提交后自动触发测试并生成报告。
    *   **产出**：这个项目是你**简历的核心**，也是你向面试官证明你“能干活”而不是“只会学”的唯一证据。

2.  **简历优化与面试准备**
    *   **简历**：将你的项目经验详细、量化地写在简历上，突出你使用的技术栈（Pytest, Selenium, Requests, Jenkins等）。
    *   **面试题库**：刷测试理论、Python编程、Linux、数据库、自动化框架设计思路等相关的面试题。
    *   **算法准备**：刷 LeetCode 上的简单和中等难度的题目，特别是字符串和列表相关的，很多公司笔试会考。
    *   **项目面试**：准备好详细介绍你的项目，包括为什么这么设计、遇到了什么困难、怎么解决的。

### 心态与建议

*   **先精后广**：不要试图同时学习所有东西。先把Python基础和Web自动化+接口自动化+Pytest这个“黄金组合”学透，再扩展其他。
*   **多写多练**：代码能力是敲出来的，不是看出来的。学习路线中的每一个技术点，都要亲手写代码实现。
*   **学会Debug**：遇到问题不要慌，学会看报错信息，利用搜索引擎和print/debug工具是工程师的核心能力。
*   **利用已有优势**：作为社招转行，你在沟通、业务理解、项目管理等方面的经验都是加分项，面试时要善于展示。
